title: 冷到热的UI
date: 2025-11-05

---

大体上可以将 UI 从冷到热进行分类：

1. 最冷的，就是 API 接口，接近于无 UI，可以利用工具来发送请求和接收返回数据。
2. 然后是凉 UI，即 CLI，用文本构建更复杂一些的，是 TUI，有输入才有输出。
3. 然后是温 UI，主要是内容为主的 Web 页面等，有较少的动画和特效，一般是保留模式，事件驱动更新。
4. 然后是热 UI，比如游戏 GUI，还有曾经的 Flash 做的 UI，即时模式，每帧更新。实时矢量绘制动画的 Flash 算是最热的 UI 之一，因为最耗计算资源和电量。
